# MUI Component Extraction

This directory contains scripts to automatically extract MUI component data from the actual MUI source code rather than hardcoding it.

## Available Scripts

### 1. `extract_mui_advanced.js` (Recommended)
Uses TypeScript Compiler API to extract complete component information including:
- Component props with types
- JSDoc descriptions  
- Required/optional status
- Default values
- Events (props starting with 'on')

**Usage:**
```bash
npm install
npm run extract
```

### 2. `extract_mui_components_auto.py`
Python script that attempts to extract data from:
- MUI GitHub repository
- TypeScript definitions
- API documentation

**Usage:**
```bash
pip install requests
python3 extract_mui_components_auto.py
```

### 3. `extract_mui_node.js` 
Simple Node.js script for runtime component inspection (limited information).

**Generated by:** `extract_mui_components_auto.py`

**Usage:**
```bash
node extract_mui_node.js
```

## Setup

1. Install dependencies:
```bash
npm install
```

2. Run the extraction:
```bash
npm run extract
```

## Output Files

- `mui_v7_components_auto_complete.json` - Complete component index
- `all_mui_components_auto.json` - All components in one file
- `[component]-mui-v7-auto.json` - Individual component files

## How It Works

1. **TypeScript Compiler API**: Parses MUI's TypeScript definitions to extract:
   - Interface definitions (ComponentProps)
   - Property types and descriptions
   - Optional/required status
   - JSDoc comments

2. **File System Scanning**: Automatically discovers MUI components by:
   - Scanning node_modules/@mui/material/src
   - Looking for directories starting with capital letters
   - Finding .d.ts or .tsx files

3. **Package.json Analysis**: Extracts component list from MUI's exports

## Advantages Over Manual Creation

- **Always up-to-date**: Extracts from actual MUI source
- **Complete type information**: Gets exact TypeScript types
- **Comprehensive**: Captures all props, not just common ones
- **Documentation**: Extracts JSDoc comments
- **Maintainable**: Re-run when MUI updates

## Limitations

- Requires MUI to be installed
- TypeScript definitions must be available
- Some runtime-only props might be missed
- Complex generic types may be simplified

## Updating for New MUI Versions

1. Update MUI version in package.json
2. Run `npm install`
3. Run extraction script again
4. New components and props will be automatically captured
